# Version v0.1.2

This release introduces a major API migration from OpenAI to Mistral AI, along with improvements to the newsletter generation workflow, including a new positive news section. The CI/CD pipeline has also been enhanced for better reliability and maintainability.

## New Features

- **Migrate to Mistral AI for newsletter generation**: The project now uses Mistral AI instead of OpenAI for all LLM-powered features, including newsletter generation. Users will need to update their API key configurations. (`aa5bf74`) (MatDagommer)
- **Add positive news section to newsletter**: Introduces a curated positive news section that appears at the top of each newsletter, sourced from selected RSS feeds. (`65e5dab`) (MatDagommer)
- **Refactor newsletter generation with structured output**: Improves the reliability and maintainability of newsletter generation by using structured data models for articles, papers, and the newsletter itself. (`6e20131`) (MatDagommer)
- **Simplify model configuration in CI**: Replaces manual CLI configuration with environment variables for setting the default language model, improving workflow reliability. (`a6c3672`) (MatDagommer)

## Bug Fixes

- **Update model configuration in release workflow**: Fixes issues with model persistence in the release workflow by using environment variables instead of direct CLI commands. (`3a1b061`) (MatDagommer)
- **Remove unnecessary horizontal rules from newsletter**: Cleans up newsletter formatting by removing redundant separators. (`70a28d3`) (MatDagommer)

## Deprecations

- **OpenAI API support**: The project no longer supports OpenAI's API. Users must migrate to Mistral AI and update their configurations accordingly. (`aa5bf74`) (MatDagommer)
